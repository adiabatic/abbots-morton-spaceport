# De-risking prototype: implementation plan

This is the week-one prototype from `doc/rebuild-design.md` §7 ("The de-risking prototype comes first"). The de-risk target is the OpenType transducer encoding — one settlement lookup of chained-context single substitutions, its compiled size, and its cross-shaper semantics — for the three worst families (qsIt, qsTea, qsMay) plus one ligature. The prototype does **not** build the new rune-file YAML format: the subset spec is hand-encoded in Python from the current `glyph_data/quikscript.yaml` facts, with provenance comments pointing at the YAML lines and at `prototype/recon/families.md`.

Ground rules (restated from the task): everything new lives under `prototype/`; no existing pipeline file, YAML, test, or site file is modified; Python runs via `uv run` only; nothing is committed or staged; `make test` stays green (trivially, since nothing it touches changes).

## 1. Module layout

All runnable modules start with `sys.path.insert(0, str(repo_root / "tools"))` (the flat-import convention from `conftest.py:18-20`) and are invoked as `uv run python prototype/<module>.py` from the repo root. Build outputs land in `prototype/out/` (untracked; never committed).

| File | Role |
| ---------------------------- | ---- |
| `prototype/spec.py` | The hand-encoded subset spec |
| `prototype/settle.py` | The pure-Python settlement function (§6.1 subset) |
| `prototype/table.py` | Decision-table enumeration + outcome-partition compression + invariants |
| `prototype/emit.py` | FEA emitter (GSUB lookups + per-height `curs`) |
| `prototype/build.py` | OTF compilation, budget measurement, extrapolation |
| `prototype/conform.py` | Exhaustive HarfBuzz conformance gate |
| `prototype/coretext_smoke.swift` | CoreText harness (new file; the recon harness stays untouched) |
| `prototype/coretext_smoke.py` | CoreText-vs-HarfBuzz driver |
| `prototype/out/` | Build artifacts: `Proto.otf`, `Proto.fea`, `settlement.tsv`, `budget.json`, conformance/smoke logs |

### spec.py

Pure data plus tiny accessors: for each of qsIt/qsTea/qsMay, the cell inventory ((entry height, exit height) tuples with anchors, bitmaps, traits, and per-cell anchor-x overrides like qsMay's en-y5 x=3-alone / x=2-combined), the pairing constraints (·It's strict en-y5⇒ex-y0 / en-y0⇒ex-y5; ·May's never-both-baseline / never-both-x-height), refusal records (today's `not_after` / `not_before` lists restricted to the subset), allowlist-polarity row scopes (today's positive `select.after` / `before` gates become `from:` / `toward:` scopes per §13.3, so absent-from-allowlist pairs stay unjoined), extension records (`extend_exit_before`, `extend_entry_after`, `extend_exit_when_entered`), the single ss03 unlock (half-Tea en-y5 after an x-height exit, plus qsMay's ss03-gated exit extension toward qsTea), and the qsTea_qsOy ligature (sequence, bitmap, exit `(8, 0)`, no entry). Every record carries a comment naming its source (`quikscript.yaml` line range from `recon/families.md` §1). Exposes the alphabet (codepoints E670/E652/E665/E679, space, U+200C) and the feature matrix `{default, ss03}`. During authoring, the implementer must run supplementary probes against today's built font (the `prototype/recon/probe_families.py` pattern) to pin the ligature's forward seams — qsTea_qsOy followed by qsIt / qsTea / qsMay — since `recon/families.md` §4 does not cover them; record the probed outcomes as provenance.

### settle.py

`settle(sequence, features) -> list[Settled]` where `Settled` carries the chosen cell (structured tuple: family, entry height or none, exit height or none, modifiers like ex-ext-1, locked flag) and the seam state toward the next position. Implements the §6.1 loop over runs (split at space; ZWNJ locks the follower's entry side and otherwise participates as a boundary token): formation first (adjacent qsTea+qsOy → ligature, left-to-right), then per-position entry binding, lookahead closure, refusals, ranking (window join-count, then structural floor), commitment. No OpenType anywhere in this module — it is the conformance oracle. Self-checks: asserts the probe rows from `recon/families.md` §4 (with the documented divergences of §7 below applied) when run as a script.

### table.py

`build_table(spec, features) -> DecisionTable`: enumerates every (settled-left context, rune, raw-right-1, raw-right-2) window reachable under settlement, calls `settle` to get the outcome cell, and partitions each slot's filler set DFA-style (two fillers are equivalent in a slot iff they yield identical outcomes for every completion of the other slots — computed by partition refinement). Emits one row per (left-class, rune, right1-class, right2-class) whose outcome differs from the input glyph (identity rows are omitted — absence of a rule is the encoding of "stay bare"). Machine-checks the §6.1 invariants on the finished table: `E-STRANDED` (every committed exit has at least one acceptor cell at the next position) and the step-4.2 joint-flag check (optimistic prospect vs. actual settled choice; divergent rows flagged `joint` in the TSV). Writes `prototype/out/settlement.tsv` (one row per transition, with provenance), the prototype-scale analog of §8's committed artifact. Also computes the reachable-cell closure — the exact glyph list `build.py` mints.

### emit.py

`emit_fea(table, spec) -> str`: produces the complete FEA string in this order — GSUB standalone lookups `proto_formation`, `proto_ss03_marker`, `proto_zwnj`, `proto_settle` (definition order fixes lookup indices, which fixes cross-feature application order), feature blocks `calt { lookup proto_formation; lookup proto_zwnj; lookup proto_settle; }` and `ss03 { lookup proto_ss03_marker; }`, then the GPOS `curs` feature with one per-height cursive lookup. Encoding details in §5 below. Asserts the §7 emitter invariant before returning: no locked twin and no chokepoint output appears in any raw lookahead class, family-keyed classes included; and every glyph named in any rule exists in the planned glyph set.

### build.py

Assembles the legacy-format `glyph_data` dict (the verified recipe from `recon/pipeline.md` §5: metadata's eight mandatory keys, `space` with `advance_width`, `uni200C` zero-advance, every reachable cell glyph as `name.prop` with its bitmap, empty `glyph_families`), calls `build_font(glyph_data, out_path, variant="senior", senior_fea=emit_fea(...))` imported from `tools/build_font.py`, then calls `_report_gsub_budget(out_path, fea_string)` (build_font.py:155) and additionally re-derives the same numbers programmatically (raw GSUB bytes via `font.reader`, per-lookup subtable byte spans) so the extrapolation arithmetic of §7d runs on numbers, not parsed stdout. Writes `prototype/out/Proto.otf`, the `.fea` sidecar (written automatically by `build_font`), and `prototype/out/budget.json` with the measured and projected figures plus the kill-criterion verdicts.

### conform.py

The exhaustive HarfBuzz gate: enumerates **all** strings of length 1–4 over the six-symbol alphabet (6 + 36 + 216 + 1296 = 1,554 sequences), shapes each through `prototype/out/Proto.otf` with uharfbuzz under both feature configurations (3,108 shaping calls), and diffs against `settle.py` transition by transition: glyph-name sequence exactly (names resolved via `TTFont.getGlyphName(info.codepoint)`, never `glyph_to_string`), then positions — accumulated pen positions must satisfy the anchor arithmetic (gap 0 at every settled join; zero advance and no ink at every ZWNJ slot). First divergent position per failing sequence is reported. Exit code nonzero on any divergence.

### coretext_smoke.swift / coretext_smoke.py

The Swift harness is the recon harness (`prototype/recon/coretext_shape.swift`) copied to `prototype/coretext_smoke.swift` and extended with an optional feature argument that sets `kCTFontFeatureSettingsAttribute` using `kCTFontOpenTypeFeatureTag`/`kCTFontOpenTypeFeatureValue` (the extension point `recon/shapers.md` §2 names) so ss03 runs are possible; it keeps the per-run resolved-font-name column so silent fallback is caught. The Python driver compiles it once per session with `swiftc -O` into `prototype/out/`, runs the curated sequence set (§7b), converts CoreText points to font units (× upem / pointSize, rounded), and diffs GID-for-GID and position-for-position against HarfBuzz on the same font. Codepoints are passed to the binary as hex arguments, never as literal PUA characters.

## 2. Ligature choice: qsTea_qsOy

Following `recon/families.md` §3's recommendation. Rationale: (a) its lead component qsTea is inside the family set, so formation interacts with the subset's own stances (the ss03-vs-formation ordering question becomes testable); (b) it is the entryless ligature, so it forces predecessor withdrawal — §5.7's exact story — which is the most interesting settlement seam available: before committing anything at position i−1, lookahead closure must see that the ligature offers no entry and land the predecessor in its withdrawn cell (·It bare, ·May bare or en-y0.ex-noentry). qsOut_qsTea is the weaker pick: its interesting edge (the after-·See bound shape) needs qsSee, outside the subset, and its entry side is an ordinary inherited (0,0). Cost accepted: formation input requires U+E679 (·Oy) in the alphabet; qsOy itself is modeled as an inert rune (bare glyph only, no entry/exit cells — it appears only when formation is suppressed).

## 3. Settlement semantics: what the subset exercises, and what it honestly does not

Exercised by real data in this subset (each with its witness):

- **Entry binding** (§6.1.1): Tea·It·May — ·It entered at y5 must exit at y0 (the `pairings: only:` rule), so the committed left seam dictates the candidate set at the next position.
- **Lookahead closure** (§6.1.2): predecessor withdrawal before entryless qsTea_qsOy (rows 23–25); It·It, where the ex-y5 candidate dies because the only would-be acceptor cell is refused (`not_after qsIt`); May·May·May's second seam; every exit candidate toward a ZWNJ-locked follower.
- **Refusals, absolute** (§6.1.3): May·It joining at x-height because `exit_baseline` carries `not_before qsIt`; Tea·Tea and It·Tea never joining.
- **Allowlist polarity** (§13.3's `from:`/`toward:` preservation): qsMay's en-y5 row is scoped to its `after:` set, which contains nothing in this subset — so It·May and Tea·May land on en-y0, not en-y5, purely by row scope, and absent-from-allowlist pairs stay unjoined instead of flipping to joined under join-count.
- **Window join-count** (§6.1.4.2): Tea·It — taking half-Tea realizes the seam and the half's prospect, beating bare Tea; the May·Tea·It ss03 chain, where the ranking between "realize the left seam with the exitless half" and "stay bare and keep the Tea·It prospect" is decided deliberately (we match today's outcome: left seam wins; the joint-flag check marks this row).
- **The structural floor** (§6.1.4.5): record-free ties settle on realize-left-seam, then lower height, then row order — exercised wherever no prefer exists, which in this subset is everywhere (see out-of-scope below).
- **E-STRANDED as a table invariant** (§6.1.2): checked over the emitted table; today's tolerated dangling exits (It·It, Tea·It·Tea) become divergences, not violations, because settlement never commits them in the first place.
- **Boundary semantics** (§3.4 via §6.1): word position falls out of run splitting (space) and the ZWNJ entry lock; word-final defaults are just rows matched when the right is a boundary.

Honestly out of scope for the prototype (REPORT.md must list these verbatim):

- **Absolute and yielding prefers** (§6.1.4.1, 4.3) and the full **extensional specificity order** (§6.2): the subset has no competing prefer records; the few narrow-vs-broad overlaps (contract-vs-extend style) are resolved by hand in `spec.py`.
- **`E-INCOMPARABLE` / `E-AMBIGUOUS` arbitration and `resolve` records** (§6.2): nothing in the subset produces them.
- **Positive `word: final` records** and the substitute-then-revert reserve encoding (§7 table row "Word position") — only fallback-row ordering is exercised.
- **Multi-set composition and composite markers**: ss03 is the only set with any effect inside the subset (`recon/families.md` §2), so the `{default, ss03}` matrix is complete but proves nothing about union semantics.
- **Bound shapes / `bind:`** (the qsOut_qsTea after-·See case) — excluded with the ligature choice.
- **Late formation on settled classes** (§7's named contingency) — formation here is unconditional and first.
- **The §9 defect suite** beyond `E-STRANDED` and gap-0 spot checks in `conform.py` (no `E-DANGLE`, `E-UNREALIZED`-as-gate, off-anchor contact, collision detection).
- **Kerning, ss06/ss10 taste overlays, the namer dot, mark/ccmp** — omitted from the prototype font entirely.

## 4. The emitter encoding (§7 table, transcribed)

GSUB lookup order (definition order = index order = cross-feature application order in both HarfBuzz and CoreText, which is itself one of the things the smoke tests confirm):

1. **`proto_formation`** — one GSUB type-4 lookup, first in `calt`: `sub qsTea qsOy by qsTea_qsOy;`. Default lookup flags (ZWNJ natively blocks ligature formation in HarfBuzz; the length-≤4 enumeration covers `qsTea ZWNJ qsOy` to verify rather than assume, in CoreText too).
2. **`proto_ss03_marker`** — referenced from `feature ss03`: `sub qsTea by qsTea.ss03;`, an unconditional marker substitution per §7 ("staged after formation and before settlement, so enabling a set cannot un-form a ligature"). `qsTea.ss03` is a bitmap-identical marker twin; the settlement classes treat it as a raw input whose row set includes the ss03-only half-Tea upgrade. Because the marker is unconditional, the ss03 gate's missing-ZWNJ-guard leak (today's `qsMay ZWNJ qsTea` + ss03 bug, `recon/families.md` §2) is structurally impossible at this stage.
3. **`proto_zwnj`** — the proven chokepoint, copied from today's encoding shape (generated FEA lines 1594–1599, emitted at quikscript_fea.py:2503-2519): `sub uni200C @proto_entry_live' by @proto_entry_locked;` where the live class is {qsIt, qsTea, qsMay, qsTea.ss03} (entry-bearing bare/marker runes; qsTea_qsOy and qsOy have no entry and need no twin) and the locked class is their `.noentry` twins (qsMay.noentry keeps the live (5,5) exit, matching today). This lookup is the only one allowed to match across ZWNJ (today's `_ZWNJ_FIREWALL_EXEMPT_LOOKUPS`, quikscript_fea.py:1881).
4. **`proto_settle`** — **one** chained-context lookup of single substitutions, the §7 settlement row shape exactly: `sub <settled-left class> <rune>' <raw class> <raw class> by <cell glyph>;`. Backtrack classes contain _settled_ glyphs (cells already substituted by earlier rules in this same lookup — GSUB backtrack sees post-substitution glyphs; this within-lookup sequencing is precisely the semantics under cross-shaper test). Lookahead classes contain _raw_ glyphs: bare runes, the marker twin, the formed ligature — and, per the emitter invariant, **never** locked twins or chokepoint outputs. Positive rules only; **zero selection-semantics `ignore sub`**. Rules are ordered most-specific-window first within each input rune (two-slot lookahead rows, then one-slot, then the bare fallback rows), and rows whose outcome equals the input glyph are simply not emitted. The settlement input domain includes the locked twins (a locked rune's exit side still settles: `ZWNJ qsTea qsIt` must yield `qsTea.noentry.half.ex-y5 | qsIt.en-y5.ex-y0`), whose outputs are `.noentry` cell twins minted only when reachable.

**Class partitioning (DFA-style outcome partition).** `table.py` computes, per input rune and per slot, the coarsest partition of that slot's filler set such that two fillers are interchangeable (identical outcome for every completion of the other slots — partition refinement until fixpoint). The emitter declares one named FEA class per partition block (`@s_<rune>_bk0`, `@s_<rune>_la1` …, names display-only) and emits one rule per (left-block, rune, right1-block, right2-block) with a distinct outcome. This is the compression whose effectiveness the budget measurement evaluates; the rule count before and after partitioning both go into `budget.json`.

**ZWNJ / default-ignorable coverage.** Primary defense is structural: the chokepoint runs before settlement, so an entry-bearing glyph after ZWNJ is already a locked twin, and locked twins appear in no raw lookahead class — a join row cannot match across ZWNJ on the right because the skipped-to glyph is not in the class. On top of that, per §7, boundary-outcome rows carry `uni200C` **explicit in the class at the boundary slot** (e.g. `[uni200C space]` in the lookahead position of every "the right is a boundary" row), ordered ahead of any join row that could match across a skipped ZWNJ, for backtrack, first, and second lookahead slots alike. The sanctioned fallback, if positive ordering proves insufficient in practice, is generated per-rule guard statements in today's proven shape — `ignore sub TARGET' uni200C;` and the two-position form `ignore sub TARGET' MID uni200C;` — copied from `_ensure_zwnj_coverage_for_calt_lookups` (quikscript_fea.py:2091) and `_add_zwnj_guards_for_two_position_forward_rules` (quikscript_fea.py:2166), with `_strip_post_zwnj_from_ignore_contexts` (quikscript_fea.py:1934) as the reference for why locked twins must not sit in ignore lookaheads. These guards are exempt from the zero-`ignore sub` claim, exactly as §7 scopes it; `budget.json` records how many were needed (target: zero).

**GPOS.** Per-height `curs` lookups in today's verbatim shape (generated FEA lines 249/706/1153/1174; emitter at quikscript_fea.py:6514-6565): `pos cursive <glyph> <entry> <exit>` with `<anchor NULL>` for the missing side, one statement per glyph per height it touches, cross-height cells appearing in both lookups with the other side NULLed, coordinates = glyph-space pixels × 50. The subset touches heights 0, 5, and 8 (qsTea's en-y8 `entry_curs_only` anchor), so three lookups are emitted; y6 has no members and is omitted (noted in REPORT.md as a naming-parity detail, not a deviation). Entryless twins with no exit are still registered `<anchor NULL> <anchor NULL>` for coverage parity with today. No `lookupflag` anywhere (today's FEA has zero in 30,838 lines).

## 5. The font build

Imports from the existing pipeline (read-only reuse, nothing modified): `build_font` and `_report_gsub_budget` from `tools/build_font.py`. The verified recipe is `recon/pipeline.md` §5: with `glyph_families` empty and `senior_fea` non-None, the IR emitter never runs and the hand-built FEA compiles as-is via `addOpenTypeFeaturesFromString`.

Glyph inventory (computed by `table.py`'s reachable-cell closure, not hand-listed; the expected shape):

- `space` (mandatory, with `advance_width`) and `uni200C` (zero advance, empty bitmap) — punctuation.yaml parity.
- Bare runes `qsIt`, `qsTea`, `qsMay`, `qsOy` (cmap via `postscript_glyph_names.yaml` through `_resolve_codepoint`, build_font.py:228), plus `qsTea_qsOy` and the marker twin `qsTea.ss03` (dotted names get no cmap entry — correct, shaping-only).
- Locked twins: `qsIt.noentry`, `qsTea.noentry`, `qsMay.noentry`, `qsTea.ss03.noentry`.
- Every cell glyph reachable under settlement across both feature configurations, including `.noentry` cell twins reachable from locked inputs and the extension cells (`qsIt.en-y5.ex-y0.ex-ext-1`, `qsMay.en-y0.ex-y5.en-ext-1`, `qsMay.ex-ext-1`, …). Bitmaps come from `spec.py` (today's shapes verbatim, including qsMay's four hand-drawn shape combinations and half-Tea's top-4-rows bitmap); 6 rows for qsIt, 9 for qsTea, 9 with `y_offset: -3` for qsMay, uniform row widths, ink rows padded with trailing spaces.

Each glyph record goes in the legacy `glyphs:` dict with a `.prop` suffix (stripped by `prepare_proportional_glyphs`, glyph_compiler.py:46). Anchors are not carried on legacy records, so `emit.py` writes every entry/exit into the `curs` feature directly, in the same coordinate frame as the drawn ink — accounting for the centering offset (`x_offset = (advance_width − bitmap_width) // 2`; with the default `width+2` advance and Senior's 1-pixel right-sidebearing shave, the left offset is exactly 1 pixel = 50 units) or passing explicit `advance_width` per glyph to make the frame trivial. `build.py` then runs `_report_gsub_budget(out_path, fea_string)` — the same call site shape as build_font.py:1212-1213 — capturing GSUB bytes / lookups / subtables, the uint16 offset headroom (the kill-criterion number), and the largest-lookup report.

### (c) DirectWrite — documented deferral

Not runnable on this Mac, and Wine's dwrite is a reimplementation that proves nothing (`recon/shapers.md` §3). REPORT.md carries the written caveat verbatim in spirit: within-lookup sequential substitution (backtrack-sees-settled) is OpenType-spec-mandated behavior that the shipping 30,838-line font already exercises against DirectWrite users today, and HarfBuzz + CoreText agreement covers two independent implementations of the contested semantics. The follow-up that closes the gap is filed in REPORT.md as a concrete next step: a `windows-latest` CI job running a ~60-line harness around `IDWriteTextAnalyzer::GetGlyphs` / `GetGlyphPlacements`, asserting the same GID/position table as (b), with ZWNJ cases 1 and 3 from `recon/shapers.md` §4 as the priority assertions.

### (d) Budget measurement, extrapolation, and the kill criteria

Measured by `build.py` from `_report_gsub_budget` plus direct table parsing: settlement rule count after partitioning (S₃), settlement rule count before partitioning (the compression ratio), settlement lookup subtable bytes (B₃), bytes per rule (b = B₃ / S₃), class-definition bytes, total GSUB bytes, LookupList and subtable uint16 offset headroom, and total FEA line count.

Extrapolation arithmetic, with assumptions stated:

- **A1 (density):** qsIt, qsTea, qsMay are three of the four worst accretion offenders (§13.6: 42 stances across the worst four), so measured rules-per-rune (S₃ / 3) is an upper-quartile density estimate; linear scaling in rune count is therefore not optimistic for the median rune, but the full alphabet's _context_ sets are richer, so a ×1.5 pessimism factor is applied on top.
- **A2 (rule count):** S₄₅ = (S₃ / 3) × 45 × 1.5.
- **A3 (bytes):** B₄₅ = S₄₅ × b + projected ClassDef bytes. ClassDef projection: measured ClassDef bytes scaled by glyph-count ratio, bounded above by the format-1 worst case of ~6 bytes per covered glyph over the full ~1,150-glyph set (~7 kB ceiling).
- **A4 (partitioning):** the per-family **subtable** split (the §7 size valve — subtables share the lookup's single left-to-right pass, so backtrack-sees-settled survives) distributes rules across 45 subtables; the binding constraint is the _largest single family's_ subtable plus shared class definitions against the uint16 subtable-offset wall, plus the cumulative subtable-offset positions within the one lookup. Projected largest family = densest prototype family's rule count × 1.5, at b bytes per rule. The cumulative-offset check: subtable offsets are measured from the Lookup table, so the _sum_ of all subtable sizes ahead of the last subtable must stay under 65,535 — if Σ projected subtable bytes > 65,535, the Extension-lookup (GSUB type 7) escape hatch exists and is noted, but needing it counts against the primary path in the report.

**Kill criteria (numeric; any one tripped ⇒ kill the single-lookup primary path and fall back to the §7 two-strata staged-wave design):**

1. **K1 — rule count:** projected full-font settlement rules S₄₅ > **10,000** (the design budget is "settlement in the low thousands of rules", §7; 10k is the hard ceiling at 2–3× that budget).
2. **K2 — offset headroom:** projected largest per-family settlement subtable (rules × b + shared ClassDef share) > **49,151 bytes** — i.e. less than **16,384 bytes** of uint16 offset headroom remaining — _or_ Σ projected subtable bytes for the settlement lookup > **65,535** with the Extension escape hatch judged unacceptable. (Stated as headroom because that is the number `_report_gsub_budget` prints.)
3. **K3 — semantics (non-numeric but binding):** any HarfBuzz-vs-settle or CoreText-vs-HarfBuzz divergence attributable to within-lookup sequential-substitution semantics or default-ignorable handling that cannot be fixed inside the §7 encoding (positive boundary-row ordering plus the sanctioned generated guards) ⇒ kill.

Secondary indicator, reported but not a kill: projected total FEA beyond the §7 envelope of ~3–6 k lines (settlement share of it) triggers a written explanation in REPORT.md. LookupList offset headroom is reported for completeness but cannot bind (the full design uses a few dozen lookups, not hundreds).

## 7. Deliberate divergences from today's font (the prototype's treaty-diff register)

`conform.py`'s oracle is `settle.py`, so these are not test failures; they are recorded in REPORT.md as intended divergences, each mapping to a §6.1 rule:

1. **It·It** (row 9): first ·It settles bare instead of carrying today's benign dangling ex-y5 — lookahead closure forbids exits with no refusal-aware acceptor (`E-STRANDED` semantics).
2. **Tea·It·Tea / Tea·It·Tea·It** (rows 11/15's middles): entered ·It before an entryless follower settles in its exit-withdrawn (en-y5, no exit) cell instead of keeping the dangling (1,0) anchor — same rule.
3. **May·It·May** (row 12): settled one way — the entered middle's `extend_exit_when_entered` carries the seam's single extension pixel and the follower's en-ext-1 is suppressed when the predecessor exit is already extended, making May·It·May's right seam identical to Tea·It·May's. Today's both-sides-summed outcome is a cascade-order artifact (`recon/families.md` §5); the settlement function must pick one, and this pick matches the design's "same-seam extensions don't sum" intent.

## 8. Failure-mode notes for the implementer

- **Lookup flags:** today's 30,838-line FEA contains zero `lookupflag` statements; ZWNJ is handled by coverage, not `IgnoreLigatures`/`UseMarkFilteringSet` flags. Do not introduce flags — they are untested territory across the three shapers and would invalidate the "verbatim today's proven shape" claim.
- **Glyph name length:** HarfBuzz's `glyph_to_string` truncates at 63 bytes and `tools/shape_sequences.py` inherits the bug — always resolve names via `TTFont.getGlyphName(gid)` (the `shaped_glyph_name` pattern, test/test_shaping.py:376-378). Keep generated cell names short regardless; identity is the structured tuple, names are display-only.
- **cmap for PUA:** `_resolve_codepoint` maps `qsX` through `postscript_glyph_names.yaml` and `uniXXXX` by hex; dotted variant names correctly get no cmap entry. PUA literals do not reliably survive shell quoting — pass hex codepoints to the Swift harness on argv (this bit recon once).
- **curs across NULL anchors:** a cross-height cell must appear in _both_ per-height lookups with the missing side `<anchor NULL>`; entryless twins with no exit still get `<anchor NULL> <anchor NULL>` registrations for coverage. Anchor coordinates are pixels × 50 in the _drawn_ frame — include the ink-centering offset or set explicit advances, or every seam will be off by exactly 50 units.
- **HarfBuzz skipping ZWNJ in context matching:** the live `qsMay ZWNJ qsTea` + ss03 leak is the proof this is real. Defenses, in order: chokepoint before settlement (locked twins not in any input or lookahead class), explicit `uni200C` boundary rows ordered first, generated ignore-guards as the sanctioned fallback. Never put a locked twin in a lookahead class (it would re-open the hole `_strip_post_zwnj_from_ignore_contexts` closes today).
- **feaLib is loud:** any glyph named in a rule or class that is missing from `glyphs:` fails the whole compile. `emit.py`'s name-existence assertion runs before `build_font` to give a better error.
- **build_font gotchas:** `space` with `advance_width` is mandatory (KeyError otherwise, build_font.py:941-942); all eight metadata keys are direct subscripts; `qs_` bitmaps must be 6 or 9 uniform-width rows, and `y_offset: -3` (qsMay) requires 9; `variant="senior"` only — never `"mono"`; Senior shaves 1 px off every `qs_` glyph's right sidebearing (`senior_tighten`, build_font.py:1032-1048).
- **uharfbuzz buffer reuse:** materialize `glyph_infos`/`glyph_positions` into lists before the next shape call, or use a fresh `hb.Buffer()` per call (do the latter; smoke-test scale doesn't need the cache). Feature dicts: `{"ss03": True}`.
- **Do not emit identity substitutions:** `sub x by x` is at best wasted budget; "stays bare" is encoded by emitting no row, which is also what keeps S₃ honest.
- **CoreText silent fallback:** assert the per-run PostScript name column on every output line; any character outside the cmap silently swaps in a system font and turns the GID diff into noise.
