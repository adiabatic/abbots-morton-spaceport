    Keys

    aSkip (explicit no-verdict record), then advance to the next unverdicted unit
    sReject — opens a follow-up choice: s no comment, a note “the old way seems nicer to write out by hand”, f note “the new way is broken”, z note “new way has a worse-looking extension/contraction”, x write a comment now; 1–9/0 reuse one of this sitting’s last ten distinct reject comments, newest first (listed below the canned choices); Escape cancels. Then advance
    eIdentical — the highlighted portion looks identical (the flagged difference is invisible), then advance
    dFine either way (any-of), then advance
    cNeither — both behaviors look wrong. Opens a follow-up choice: c no comment, d note “Under ss10 these must be fully isolated; old font joins them, new font ligates them — both wrong”, x write a comment now; 1–9/0 reuse one of this sitting’s last ten distinct neither comments, newest first; Escape cancels. Then flag for follow-up authoring and advance
    fApprove, then advance
    uUndo the last verdict and return to it
    rRepeat — apply the previous verdict and its note (as it stands now, edits included) to the current unit, then advance. Never toggle-clears, so it’s safe to hammer down a run of identical screwups
    ⌫Clear the current unit’s verdict (Delete works too; pressing its active verdict key again also clears)
    nFocus the current unit’s note field; Enter in the note saves and advances, Shift+Enter saves and stays (so you can verdict with the note filled in). The field offers this sitting’s previous comments as you type — pick one to reuse its exact wording
    gApprove every remaining unverdicted unit in the current group
    xToggle the current unit’s explain/provenance panel (same as the row’s “Why?” button)
    ↓ / ↑Move the cursor without verdicting
    k / iSame as ↓ / ↑ — next / previous unit
    [ / ]Previous / next class in the sidebar list (wraps; skips classes with no human workload)
    /Focus the cross-class unit search (matches id, notation, codepoints, and class/group/kind across every class); ↓/↑ move, Enter jumps, Escape closes
    ?This overlay
    EscapeBlur input / close overlay

    Verdicting a unit that has no verdict yet advances to the next unverdicted unit; re-verdicting a unit that already has a verdict updates it in place and keeps the cursor put, so you can revise without losing your spot. The “First unverdicted” button up top jumps to the first unverdicted unit anywhere on the surface — scanning every batch in order across every class, switching batch (and clearing the class filter if needed) to land on it.

    The small text box left of “First unverdicted” previews arbitrary Quikscript in the rebuild (after) font. Type letter names joined with middots (·day·utter; plain dots work too), separate words with spaces, and use zwnj for a non-joiner or uniXXXX (or a bare PUA hex like e653) for a raw codepoint — pasted Quikscript characters render as-is. It renders with kerning off and default features, matching the After column; clear the box to dismiss the preview.

    Most units diverge under every non-ss10 config and carry no config marker; a unit shows a small badge (“only when ss03 is on”, “only under ss10”, …) only when its config set is informative — hover it for the full list. A verdict always covers all of the unit’s configs. If a unit ever has configs that render differently, each distinct rendering is stacked below the first with its own label.

    Units whose ink is pixel-identical in both fonts under every config (or, for ss10-only divergences, pixel-identical to the Junior font’s isolated rendering) were machine-approved at build time, and units in a no-verdict ledger class (like boundary-echo, whose windows the ratified boundary rule adjudicates wholesale) never need individual verdicts; both kinds are hidden by default. The “Show no-verdict units” toggle reveals them, badged and with verdict controls disabled — batches, progress counts, and the verdict keys cover the human workload only.

    Both columns render with kerning disabled (font-kerning: none), and every build-time comparison shapes with kern off: the rebuild has no kerning until its own later milestone, so the old font’s kern feature would be pure comparison noise. Kern differences get their own review when that milestone lands.

    Unmarked regions render identically in both fonts (or differ only invisibly, at name grain); a dim dashed band marks a secondary divergent seam that is judged at its linked home unit — click the band’s chip to jump there (“only here” means the seam has no shorter home and is judged in this unit).

    An “echo ×N” chip means the unit’s before→after change is pixel-identical in N windows of the same class with the same judged pair — the surrounding letters differ but the change is the same picture, so it is one question. A verdict (other than skip) on any member fills the unverdicted rest, with the toast confirming the count and u undoing the whole fill; already-verdicted members are never overwritten, and each member can still be overridden or cleared individually. Click the chip to view the whole echo group stacked as a worklist.

    The “Docket” button opens a live work order over the blank (unverdicted or skipped) units: clusters that share one before→after ink delta across echo groups, largest first, then singletons and any echo groups whose recorded verdicts disagree. Every button stacks a decision as a worklist in this app, and the queue recomputes as verdicts land — clusters shrink, empty, and promote without any re-bake. Judging the last unit of a docket worklist advances straight to the next decision in queue order (skipped units count as conscious deferrals and are never re-stacked), the progress line up top tracks the blanks remaining, and the strip above the rows links back to the full queue. Typing an echo-group id (like e-0061) into the search offers the whole group stacked as a worklist.

    Verdicts autosave to the server after every change, and every open copy of the app re-syncs from that autosave whenever its window regains focus — so a docket or batch left in another tab picks up verdicts landed elsewhere on its own (the newer record per unit wins, same as import). Clearing a verdict doesn’t cross tabs: another still-open copy may re-save it, so do any clearing in the tab you’re actively judging in.
