    About this page

    Generated review surface for the rebuild treaty diff. Regenerate with and serve with (port 7294, alongside make serve on 7293). Verdicts live only in this tab until you export them.

    Keys

    aSkip (explicit no-verdict record), then advance to the next unverdicted unit
    sReject — opens a follow-up choice: s no comment, a note “the old way seems nicer to write out by hand”, f note “the new way is broken”, x write a comment now; Escape cancels. Then advance
    dFine either way (any-of), then advance
    cNeither — both behaviors look wrong; flag for follow-up authoring, then advance
    fApprove, then advance
    uUndo the last verdict and return to it
    nFocus the current unit’s note field; Enter in the note saves and advances
    gApprove every remaining unverdicted unit in the current group
    xToggle the current unit’s explain/provenance panel (same as the row’s “Why?” button)
    ↓ / ↑Move the cursor without verdicting
    k / iSame as ↓ / ↑ — next / previous unit
    [ / ]Previous / next batch
    ?This overlay
    EscapeBlur input / close overlay

    Most units diverge under every non-ss10 config and carry no config marker; a unit shows a small badge (“only when ss03 is on”, “only under ss10”, …) only when its config set is informative — hover it for the full list. A verdict always covers all of the unit’s configs. If a unit ever has configs that render differently, each distinct rendering is stacked below the first with its own label.

    Units whose ink is pixel-identical in both fonts under every config were machine-approved at build time and are hidden by default; the “Show machine-approved” toggle reveals them, badged and with verdict controls disabled — batches, progress counts, and the verdict keys cover the human workload only.

    Both columns render with kerning disabled (font-kerning: none), and every build-time comparison shapes with kern off: the rebuild has no kerning until its own later milestone, so the old font’s kern feature would be pure comparison noise. Kern differences get their own review when that milestone lands.

    Unmarked regions render identically in both fonts (or differ only invisibly, at name grain); a dim dashed band marks a secondary divergent seam that is judged at its linked home unit — click the band’s chip to jump there (“only here” means the seam has no shorter home and is judged in this unit).
