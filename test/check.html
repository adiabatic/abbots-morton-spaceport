Check — before/after

Side-by-side rendering harness. Three auto-generated sections cover the whole page: one renders every assertion line from currently- failing pytest tests so you can eyeball false positives; one diffs every multi-letter Quikscript run in the test corpus between the snapshot under test/before/ and the live build under test/; one lists every short sequence whose adjacent non-joining pair changes shape between single-buffer and split shaping.

Workflow:

  1. On the baseline you want to compare against (typically master), run make snapshot-before. It builds the fonts and copies all six OTFs into test/before/ (gitignored).
  2. Make your code or YAML changes on a branch.
  3. make check-html rebuilds the live OTFs, runs the pytest suite to gather failing assertions, and regenerates this file end-to-end via tools/build_check_html.py.
  4. Reload this page. Codepoint references live in reference/csur/index.html; the family-to-codepoint map is in postscript_glyph_names.yaml.
