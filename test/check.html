Check — before/after

Side-by-side rendering harness. The left column loads the snapshot under test/before/; the right column loads the live build under test/. Sequences identical between columns indicate the change you’re working on left them untouched. Anything that differs visually is the part of the change you can eyeball.

Workflow:

  1. On the baseline you want to compare against (typically master), run make snapshot-before. It builds the fonts and copies all six OTFs into test/before/ (gitignored).
  2. Make your code or YAML changes on a branch.
  3. make all to rebuild test/*.otf, then reload this page.
  4. Edit the sections below to cover whatever you’re inspecting. Codepoint references live in reference/csur/index.html and the family-to-codepoint map is in postscript_glyph_names.yaml.
