Check — before/after

Side-by-side rendering harness. Auto-generated sections cover most of the page: one renders every assertion line from currently-failing pytest tests so you can eyeball false positives; one diffs every multi-letter Quikscript run in the test corpus between the snapshot under test/before/ and the live build under test/; one lists every short sequence whose adjacent non-joining pair changes shape between single-buffer and split shaping.

Workflow:

  1. On the baseline you want to compare against (typically master), run make snapshot-before. It builds the fonts and copies all six OTFs into test/before/ (gitignored).
  2. Make your code or YAML changes on a branch.
  3. make check-html rebuilds the live OTFs, runs the pytest suite to gather failing assertions, and regenerates this file end-to-end via tools/build_check_html.py.
  4. Reload this page. Codepoint references live in reference/csur/index.html; the family-to-codepoint map is in postscript_glyph_names.yaml.

Auto-generated: isolation leaks

Sequences whose adjacent pair does not cursive-attach but whose chosen glyphs differ when the pair is shaped together versus split into independent buffers. These are the cases that currently require |?| in data-expect; visually inspect each row to decide whether the cross-break shape change is cosmetic or a real bug. Generated with --max-len 3.

Columns: the middle column shapes the whole sequence as a single buffer (what you get in real text); the right column splits the sequence at the leaky break into two display: inline-block halves so HarfBuzz shapes each side independently. Decide whether the in-context shape is intended.

Only diff rows — where the in-context buffer and the concatenation of the two independently-shaped halves differ in their per-glyph (pixels, absolute origin) sequence — are shown. Comparing absolute origins (not just pixels) catches cursive-positioning leaks: e.g. qsIt vs qsIt.ex-y5 have identical bitmaps but the latter's exit anchor pulls the next glyph leftward via GPOS curs. Pure glyph-name-signature changes with no visible effect (the old same rows, typically after-tall-style trims) are filtered out.
